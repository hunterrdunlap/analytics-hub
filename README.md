# Analytics Hub

Internal analytics portal for the NFS team. Tracks projects across three divisions (Reinsurance, Real Estate, Structured Finance) with reports, dashboards, documents, and oversight controls.

## Architecture

- **Frontend**: Vanilla JS single-page app â€” no build tools, no framework. Served via GitHub Pages.
- **Backend**: AWS Lambda (Node.js 20.x) + API Gateway HTTP API + DynamoDB (serverless).
- **Infrastructure**: Terraform (in `terraform/`) with modules for database, lambda, and API.

## Project Structure

```
index.html              # App shell
js/
  config.js             # API_BASE_URL (auto-generated by deploy.sh)
  api.js                # Fetch wrapper with TTL cache + request coalescing
  data.js               # DataStore â€” async data access layer
  router.js             # Client-side routing and state management
  app.js                # App shell, sidebar, modals, loading indicators
  zones.js              # Zone renderers (Reports, Performance, Controls)

  projects/             # CRUD for projects
  divisions/            # Division lookup
  reports/              # Reports CRUD
  documents/            # Documents CRUD
  dashboard-links/      # Dashboard links CRUD
  control-items/        # Control items CRUD
  requests/             # Requests CRUD
  in-progress/          # In-progress items CRUD
terraform/              # IaC â€” modules/database, modules/lambda, modules/api
scripts/
  deploy.sh             # Terraform apply + updates config.js with API URL
  seed-data.js          # Migrates localStorage export into DynamoDB
```

## Data Model

- **3 divisions** (hardcoded): Reinsurance, Real Estate, Structured Finance
- **7 DynamoDB tables**: projects, requests, in-progress, reports, documents, dashboard-links, control-items
- All tables use `id` as partition key with a GSI on `projectId` (or `divisionId` for projects)
- ID prefixes: `proj`, `req`, `prog`, `rpt`, `doc`, `dash`, `ctrl`

## Deploying

```bash
./scripts/deploy.sh
```

This runs `terraform apply` and writes the API Gateway URL into `js/config.js`. The frontend is static â€” push to `main` and GitHub Pages picks it up.

## Local Development

Open `index.html` directly in a browser. The app will call the deployed API â€” no local server needed.

To seed data from a localStorage export:

```bash
node scripts/seed-data.js <export-file.json>
```
